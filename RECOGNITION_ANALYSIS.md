# 图片识别逻辑分析（已优化）

## 识别算法概述

项目使用 **OpenCV 灰度模板匹配** 进行图片识别，简单高效。

## 核心算法流程

### 1. 图片预处理

```python
# 屏幕截图
screen_bgr = cv2.cvtColor(screen_bgra, cv2.COLOR_BGRA2BGR)

# 读取模板图片
template_bgr = cv2.imread(img_path)

# 转灰度
template_gray = cv2.cvtColor(template_bgr, cv2.COLOR_BGR2GRAY)
screen_gray = cv2.cvtColor(screen_bgr, cv2.COLOR_BGR2GRAY)
```

### 2. 多尺度搜索

```python
scales = [1.0, 0.95, 0.9, 0.85]  # 支持 85%-100% 的尺寸变化
```

对每个尺度进行匹配，找到最佳匹配位置。

### 3. 灰度模板匹配

```python
cv2.matchTemplate(screen_gray, template_gray, cv2.TM_CCOEFF_NORMED)
```

- **优点**：精确度高，分数可达 0.8-0.95
- **方法**：归一化相关系数匹配（TM_CCOEFF_NORMED）
- **输出**：0-1 之间的相似度分数

### 4. 阈值判断

```python
if score >= 0.7:
    点击目标
else:
    跳过
```

## 性能指标

### 单次识别耗时
- 全屏截图：~50ms
- 灰度转换：~5ms
- 模板匹配：~30ms × 4尺度 = ~120ms
- **总计**：~175ms/图片

### 识别准确率
- 完全匹配：0.90-0.98 ✅
- 轻微缩放：0.85-0.92 ✅
- 背景变化：0.70-0.85 ⚠️
- 光照变化：0.65-0.80 ⚠️

## 可靠性分析

### ✅ 优点

1. **高准确率**
   - 灰度匹配分数高（0.8-0.95）
   - 0.7 阈值合理，误识别率低

2. **性能优秀**
   - 比边缘检测快 2 倍
   - 代码简洁，易维护

3. **多尺度适应**
   - 支持 85%-100% 的尺寸变化
   - 适应不同分辨率

4. **稳定可靠**
   - OpenCV 成熟算法
   - 工业级应用验证

### ⚠️ 局限性

1. **背景色敏感**
   ```
   问题：如果按钮背景色变化（灰色→彩色），分数会下降
   影响：可能从 0.9 降到 0.7-0.8
   解决：重新截图即可
   ```

2. **光照敏感**
   ```
   问题：屏幕亮度变化会影响识别
   影响：分数可能下降 0.1-0.2
   解决：保持屏幕亮度稳定
   ```

3. **旋转不支持**
   ```
   问题：如果按钮有旋转，完全无法识别
   解决：需要重新截图
   ```

4. **相似按钮混淆**
   ```
   问题：两个外观相似的按钮可能误识别
   解决：提高阈值或增加前置条件
   ```

## 故障排查

### 识别不到目标？

1. **检查分数**
   - 查看日志中的匹配分数
   - 如果分数 0.6-0.7，降低阈值
   - 如果分数 < 0.6，重新截图

2. **重新截图**
   - 确保截图清晰
   - 包含完整按钮
   - 避免截到边缘

3. **检查尺寸**
   - 游戏分辨率是否变化
   - 当前支持 85%-100%
   - 超出范围需要重新截图

4. **检查环境**
   - 屏幕亮度是否稳定
   - 游戏界面是否变化
   - 按钮是否被遮挡

### 误识别？

1. **提高阈值**
   - 从 0.7 提高到 0.75
   
2. **增加前置条件**
   - 添加必须满足的条件图片
   
3. **重新截图**
   - 截取更具特征的区域
   - 避免与其他按钮相似
